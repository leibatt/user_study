Overview

Overview

  • Leaving the User Study Website

    • Closing the user study browser window/tab or navigating away from the user study website will end your participation in the user study.
    • User/account information is not saved when you end your participation in the study. Therefore, if you revisit the user study website, a new participation session will be created and you will not have access to your previous information.

  • User Study Tasks

    • There are 2 research-related user study tasks.
    • Each task requires you to select a set of visualizations from ScalaR that meet given goal requirements.
    • The overview section of each task page describes the data set being visualized in the task, and describes the goal of the task.
    • The goal describes the requirements for the visualizations you need to find.
    • The task pages have collapsible sections, so you can hide what you don't need.
    • Hover over icons to learn more about the various menus and buttons of ScalaR while completing the tasks.

  • Making Selections

    • To select visualizations, check the checkbox just under the visualization:
    • A snapshot of each selected visualization is saved, along with the location of the corresponding data.
    • To undo your current selection, click the same checkbox to remove the check:
    • Your selections are saved, so you will not lose your answers when navigating to other web pages within the user study site.

  • Reviewing Your Selections

    • Go to the task's Selections page to review your selections.
    • The Selections page allows you to remove multiple visualization selections at once.
General Options

General Options

  • Changing the Resolution

    • ScalaR lets you specify an upper bound on how many data points are visualized from the data set.
    • ScalaR needs more time to retrieve and draw more data points, but the resulting visualizations are more detailed.

  • Starting Over

    • To quickly return to the overview visualization of the data set (the first visualization you see), click on the "Update Resolution" button. Make sure the resolution menu is set to your desired number of data points.
    • Use the "Reset Answers for [Task]" button to quickly remove all visualization selections for the current task. You can also do this on the task's Selection page.
Exploring the Data Set and Visualizations

Exploring the Data Set and Visualizations

  • Directional Buttons and Click Zoom

    • There are 4 directional buttons to move up, down, left, and right through the data set:
      • Moving up/down tells ScalaR to retrieve and visualize neighboring data in the y-dimension (x-axis range remains the same).
      • Moving left/right tells ScalaR to retrieve and visualize neighboring data in the x-dimension (y-axis range remains the same).
    • ScalaR has click-zoom functionality:
      • Left-Click on the visualization to zoom in to the area of the visualization that was clicked on.
      • Right-click on the visualizatoin to zoom out.

  • Reading the Visualization

    • Each visualization in ScalaR is generated from a specific subset of data from the database called a "data tile".
    • Each "data tile" is in the form of a 2-dimensional array. Each cell can contain multiple values called attributes.
    • In this user study, all visualizations are heatmaps. Each rectangle in the heatmap corresponds to one cell in the data tile.
    • The axes of the heatmap are the dimensions of the data tile.
    • The colors of the heatmap are dictated by either cell attributes or the dimensions from the data tile.
Visualization Options

Visualization Options

  • Changing the Axes

    • The "xAxis" and "yAxis" drop-down menus tell ScalaR what attributes from the data set should be the x-axis and y-axis of the visualization.
    • The "zAxis" drop-down menu tells ScalaR what attribute from the data set should be used to generate the color scale for the visualization.
    • The "invert" radio buttons tell ScalaR invert the mappings of their respective axes:
      • x-axis: data points mapping to the left of the visualization now map to the right of the visualization and vice versa.
      • y-axis: data points mapping to the bottom of the visualization now map to the top of the visualization and vice versa.
      • z-axis: data points now map to their opposite color in the legend.

  • Changing the Color Scheme

    • The color scheme drop-down menu tells ScalaR what colors to use in the visualization.

  • Resizing the Visualization

    • The width and height text boxes tell ScalaR what dimensions to use to draw the visualization.

  • Updating the Visualization With Your Changes

    • Click on the "Submit" button at the bottom of the visualization options menu to update the visualization with your changes.
